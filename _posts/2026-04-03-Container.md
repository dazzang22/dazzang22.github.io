## Container

Container 란 ? 

여러 개의 데이터 객체를 **하나로 묶어서** 저장, 관리, 구조화하는 데이터 구조. (-> **데이터를 담는 용기**)

삭제, 삽입, 검색 기능을 제공한다.

종류 : Vector, Array, List

## Vector 

**연속된 메모리**를 사용하는 동적 배열로, array와 다르게 컴파일타임에 크기를 정하지 않아도 된다. Heap Memory 에 저장된다.

```c++
vector<int> v;
```

삽입, 삭제 - back: `push_back` 이나 `pop_back` 를 사용하면 O(1) 의 시간복잡도를 갖는다.

삽입, 삭제 - 중간 : O(n)의 시간복잡도를 갖는다.

정렬 : O(NlogN)의 시간복잡도를 갖는다.

접근 : 원소 한개 - O(1) 의 시간복잡도를 갖는다.

장점

* 배열처럼 접근이 빠르다. O(1)
* 순회가 빠르다.
* 캐시 효율이 좋다 - 왜? 연속된 메모리를 사용하기 때문에 순회가 빈번하게 일어나는 경우 주변의 데이터도 캐시에 올려둬 효율을 올릴 수 있다.

단점

* 중간 삽입/ 삭제가 느리다.
* 재할당 비용이 크다

언제 ? **가변**적인 데이터를 **순회**하는 로직이라면 vector 채택

## Array

연속된 메모리를 사용하는 고정 배열로, 컴파일 타임에 사이즈를 명시해야 한다. Stack 또는 정적 메모리에 저장된다.

```c++
int a[300];
```

삽입, 삭제 - 

접근 : 원소 한개 - O(1) 의 시간복잡도를 갖는다.

장점

* 접근이 빠르다.
* 순회가 빠르다.
* 오버헤드가 벡터에 비해 적다. (고정 배열이기 때문)

단점 

* 크기 재할당이 불가능

언제 ? 크기가 고정되어있는 데이터를 다룰 때 사용한다.

## List

비연속 연결 리스트로, 노드와 next를 가리키는 포인터로 이루어져있다.

접근 : O(n)

삽입, 삭제 : O(1) + 삽입할 위치를 찾는 비용

장점

* 특정 위치에서 삽입/삭제가 빠르다.

단점

* 인덱스 접근이 불가하여 순회가 느리다.
* 메모리 오버헤드가 크다.
* 캐시 효율이 나쁘다. (비연속이라)

## 정리

매 frame 마다 계산비용이 큰 게임에서는 순회가 빠르고 크기를 조정할 수 있으며 캐시효율성이 좋은 vector 를 쓰는것이 편리하고 좋다.
